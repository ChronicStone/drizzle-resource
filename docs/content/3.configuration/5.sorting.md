# Sorting

This page is about configuration, not the sorting payload the client sends.

Sorting behavior lives under `query.sort`.

## Set fallback sorting with `defaults`

```ts [resource.ts]
query: {
  sort: {
    defaults: [{ key: 'id', dir: 'asc' }],
  },
}
```

The defaults apply when `request.sorting` is empty.

## Disable unsupported sort fields with `disabled`

```ts [resource.ts]
query: {
  sort: {
    defaults: [{ key: 'id', dir: 'asc' }],
    disabled: ['employeeSkills.skill.label'],
  },
}
```

Use this for fields that remain known to the resource but should not be sortable.

## Disable sort paths when SQL would be unstable or too expensive

Common reasons:

- many-to-many relation paths
- non-deterministic expressions
- relation-heavy sorts that need a custom ids strategy

## Configure defaults and disabled fields together

```ts [resource.ts]
const employees = engine.defineResource('employees', {
  relations: {
    department: {
      with: {
        company: true,
      },
    },
    employeeSkills: {
      with: {
        skill: true,
      },
    },
  },
  query: {
    sort: {
      defaults: [{ key: 'id', dir: 'asc' }],
      disabled: ['employeeSkills.skill.label'],
    },
  },
})
```

## Expect invalid sort fields to fail fast

Unknown sort fields are rejected before execution.
