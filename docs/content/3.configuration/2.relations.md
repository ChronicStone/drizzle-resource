# Relations

`options.relations` is the Drizzle `with` tree attached to your resource.

## Declare the relation tree you want to expose

```ts [resource.ts]
const resource = engine.defineResource('employees', {
  relations: {
    department: {
      with: {
        company: true,
      },
    },
    employeeSkills: {
      with: {
        skill: true,
      },
    },
  },
})
```

## Use relations to shape the public field registry

The relation tree controls:

- which nested field paths become valid
- what the built-in rows query hydrates
- the default row type for `query(...)` and `queryRows(...)`

## Compare a narrow and wide relation tree

```ts [resource.ts]
const compactEmployees = engine.defineResource('employees', {
  relations: {
    department: true,
  },
})

const detailedEmployees = engine.defineResource('employees', {
  relations: {
    department: {
      with: {
        company: true,
      },
    },
    employeeSkills: {
      with: {
        skill: true,
      },
    },
  },
})
```

The second resource unlocks field paths such as:

- `department.company.name`
- `employeeSkills.skill.label`

It also makes those relations available to the default rows hydration.

## Know what the built-in rows query is aiming for

```ts [conceptual-hydration.ts]
db.query.employees.findMany({
  where: {
    id: {
      in: ids,
    },
  },
  with: {
    department: {
      with: {
        company: true,
      },
    },
    employeeSkills: {
      with: {
        skill: true,
      },
    },
  },
})
```

## Keep relations intentional

Only include relations that belong in the public query contract.

Adding a relation can increase:

- the number of valid queryable field paths
- hydration cost
- the chance of expensive relation-heavy sorts or facets
