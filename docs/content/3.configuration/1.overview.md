# Overview

`defineResource(root, options)` is the main surface for shaping a query resource.

## Start with the three configuration layers

Each resource is organized around:

- `root` for the base table
- `relations` for valid nested field paths and default hydration
- `query` and `strategy` for behavior

```ts [resource.ts]
const employees = engine.defineResource('employees', {
  relations: {
    department: {
      with: {
        company: true,
      },
    },
    employeeSkills: {
      with: {
        skill: true,
      },
    },
  },
  query: {
    scope: engine.defineScope(
      'employees',
      {
        department: {
          with: {
            company: true,
          },
        },
        employeeSkills: {
          with: {
            skill: true,
          },
        },
      },
      (ctx, filters) => filters.and([filters.is('department.company.country', ctx.orgId)]),
    ),
    search: {
      allowed: ['fullName', 'email', 'department.name', 'department.company.name'],
      defaults: ['fullName', 'department.company.name'],
    },
    sort: {
      defaults: [{ key: 'id', dir: 'asc' }],
      disabled: ['employeeSkills.skill.label'],
    },
    filters: {
      hidden: [],
      disabled: [],
    },
    facets: {
      allowed: ['department.name', 'department.company.name', 'employeeSkills.skill.label'],
    },
    defaults: {
      pagination: {
        pageSize: 25,
      },
    },
  },
})
```

## Keep the mental model simple

Use `query` to declare what callers are allowed to ask for.

Use `strategy` only when you need to override how the package executes that contract.
