# Pagination and Defaults

This page is about configuring defaults, not the request payload itself.

Default behavior lives under `query.defaults` and parts of `query.search` and `query.sort`.

## Configure pagination defaults

```ts [resource.ts]
query: {
  sort: {
    defaults: [{ key: 'id', dir: 'asc' }],
  },
  search: {
    defaults: ['fullName'],
  },
  defaults: {
    pagination: {
      pageIndex: 1,
      pageSize: 25,
    },
  },
}
```

## Know the pagination semantics

- `pageIndex` is one-based
- invalid `pageIndex` falls back to the configured default or `1`
- invalid `pageSize` falls back to the configured default or `25`

## Make the empty request predictable

Useful defaults usually include:

- `query.search.defaults` for empty `search.fields`
- `query.sort.defaults` for empty `sorting`
- `query.defaults.pagination` for invalid or missing pagination values

## Keep defaults aligned with the screen you are serving

Choose defaults that match the UX of the endpoint rather than aiming for one universal preset.
