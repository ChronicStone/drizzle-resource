# Filters

This page is about resource configuration, not client payload shape.

Filter policy lives under `query.filters`.

## Hide fields entirely with `hidden`

```ts [resource.ts]
query: {
  filters: {
    hidden: ['salaryInternal'],
  },
}
```

Hidden fields are removed from the field registry entirely. They cannot be:

- filtered
- searched
- sorted
- faceted

## Keep a field known but non-filterable with `disabled`

```ts [resource.ts]
query: {
  filters: {
    disabled: ['employeeSkills.skill.label'],
  },
}
```

Disabled filter fields remain known to the resource, but explicit filtering on them is rejected.

## Configure hidden and disabled fields together

```ts [resource.ts]
const employees = engine.defineResource('employees', {
  relations: {
    department: {
      with: {
        company: true,
      },
    },
    employeeSkills: {
      with: {
        skill: true,
      },
    },
  },
  query: {
    filters: {
      hidden: ['salaryInternal'],
      disabled: ['employeeSkills.skill.label'],
    },
  },
})
```
