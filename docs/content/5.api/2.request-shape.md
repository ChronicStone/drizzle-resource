# Request Shape

::note
For the user-facing, example-driven explanation of the client payload, start with [Query Contract](/query-contract/overview). This API page is the more formal reference.
::

The public request object is centered on pagination, sorting, filters, search, and optional facets.

## Send a complete request object

```ts [query-request.ts]
type QueryRequest = {
  pagination: {
    pageIndex: number
    pageSize: number
  }
  sorting: Array<{
    key: string
    dir: 'asc' | 'desc'
  }>
  filters: QueryFilterGroup
  search: {
    value: string
    fields: string[]
  }
  context: Record<string, unknown>
  facets?: QueryFacetRequest[]
}
```

## Normalize pagination before your strategy runs

- `pageIndex` is one-based
- invalid `pageIndex` falls back to the configured default or `1`
- invalid `pageSize` falls back to the configured default or `25`

## Use empty search fields to trigger defaults

When `request.search.fields` is an empty array, the package falls back to `query.search.defaults`.

When `request.search.value === ''`, search compilation is skipped.

Search matching is case-insensitive.

## Build filters as a tree

```ts [filters.ts]
{
  type: 'group',
  combinator: 'and',
  children: [
    {
      type: 'condition',
      key: 'department.company.name',
      operator: 'contains',
      value: 'labs',
    },
  ],
}
```

Supported operators include:

- `contains`
- `is`
- `isAnyOf`
- `isNot`
- `gt`
- `gte`
- `lt`
- `lte`
- `between`
- `before`
- `after`

## Expect invalid field usage to fail fast

The package rejects:

- unknown sort fields
- unknown search fields
- unknown filter fields
- unknown facet fields
