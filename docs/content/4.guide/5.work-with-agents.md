# Work with Agents

If AI tooling or automation edits your resources, give it clear guardrails.

## Preserve the package guarantees

When agents work on a resource, they should preserve these guarantees:

- public field paths stay valid and intentional
- scopes enforce tenant and auth constraints by default
- query defaults remain predictable
- custom strategies reuse built-in helpers when possible
- performance tradeoffs are documented when deviating from defaults

## Follow a safe resource workflow

Use this checklist before changing a resource:

1. Identify the root table, schema entry, and relations object.
2. Inspect the existing `relations` tree.
3. Enumerate the public query contract for search, sort, filters, and facets.
4. Add or update tests before changing strategy behavior.
5. Prefer declarative `query` config first.
6. Introduce `strategy.*` overrides only when there is a concrete SQL or shape requirement.

## Prefer the safer implementation patterns

Good patterns for automation:

- use `defineScope` for shared constraints
- override `strategy.ids` before jumping to `strategy.query`
- reuse `utils` inside custom strategies
- document performance-sensitive decisions in code or docs

## Run the minimum release hygiene checks

Before treating an agent-authored change as release-ready, run:

::code-group
```bash [bun]
bun run test
bun run typecheck
bun run build
```

```bash [npm]
npm run test
npm run typecheck
npm run build
```
::
