# Define an Engine

Use one engine per database and schema graph.

## Connect your Drizzle inputs

```ts [engine.ts]
const engine = createQueryEngine({
  db,
  schema,
  relations,
})
```

The engine needs:

- `db` for relational reads and SQL execution
- `schema` for table and column metadata
- `relations` for relation traversal and nested field inference

## Add context when your resources need it

If your resources depend on tenant or auth context, extend the engine once and reuse it everywhere.

```ts [engine.ts]
const engine = createQueryEngine({
  db,
  schema,
  relations,
}).withContext<{ orgId: string; userId: string }>()
```

## Reuse the same engine across related resources

This keeps your resources aligned on:

- root table inference
- relation graph typing
- shared scope helpers
- strategy utility behavior

## Keep the engine thin

The engine should describe infrastructure, not endpoint behavior.

Define endpoint-specific behavior later inside `defineResource(...)` through:

- `relations`
- `query`
- `strategy`
