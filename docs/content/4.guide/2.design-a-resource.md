# Design a Resource

Use `defineResource(...)` to turn one root table into a release-ready query surface.

## Declare the relations you want to expose

Your `relations` tree controls two important things:

- which nested field paths become valid
- which relations the built-in rows query hydrates

```ts [resource.ts]
const resource = engine.defineResource('employees', {
  relations: {
    department: {
      with: {
        company: true,
      },
    },
    employeeSkills: {
      with: {
        skill: true,
      },
    },
  },
})
```

## Configure the public query contract

Use `query` to define what callers are allowed to ask for.

```ts [resource.ts]
query: {
  search: {
    allowed: ['fullName', 'department.company.name'],
    defaults: ['fullName'],
  },
  sort: {
    defaults: [{ key: 'id', dir: 'asc' }],
    disabled: ['employeeSkills.skill.label'],
  },
  filters: {
    hidden: ['salaryInternal'],
    disabled: ['employeeSkills.skill.label'],
  },
  facets: {
    allowed: ['department.name', 'employeeSkills.skill.label'],
  },
}
```

## Use `defineScope` for shared constraints

Scope builders keep tenant and auth logic typed and reusable.

```ts [scope.ts]
const employeeScope = engine.defineScope(
  'employees',
  {
    department: {
      with: {
        company: true,
      },
    },
  },
  (ctx, filters) => filters.and([filters.is('department.company.country', ctx.orgId)]),
)
```

Then reuse the scope across resources or endpoint variants:

```ts [resource.ts]
query: {
  scope: employeeScope,
}
```

## Hide only what should truly disappear

Use the field-visibility rules intentionally:

- `filters.hidden` removes a field from the public registry entirely
- `filters.disabled` keeps the field known but rejects explicit filtering
- `sort.disabled` keeps the field available for search or facets while blocking sorting

## Set defaults that make the empty request predictable

Good defaults reduce client guesswork.

Configure:

- `query.search.defaults` for empty `search.fields`
- `query.sort.defaults` for empty `sorting`
- `query.defaults.pagination` for invalid or missing pagination values
