---
title: Type Safety
description: Field paths are inferred from your Drizzle schema and relation tree. The scope filter builder, context shape, and field keys are all typed — typos are compile errors, not runtime surprises.
---

`drizzle-resource` builds a typed union of every reachable field path from your resource declaration — flat columns (`status`, `reference`) and nested relation paths (`customer.name`, `orderLines.product.category`). That union flows through the scope filter builder and the query contract.

## Typed scope filter builder

The `f` argument in `query.scope` is a typed filter builder. Hover any path or method to see the constrained contract — paths are derived from your `relations` tree, not from a `string` type:

```ts twoslash
import { engine } from "./engine";
// ---cut-before---
engine.defineResource("orders", {
  relations: {
    customer: true,
    orderLines: { with: { product: true } },
  },
  query: {
    scope: (f, ctx) => f.and([f.is("customer.orgId", ctx.orgId), f.isNot("status", "archived")]),
  },
});
```

## Auto-completion across relation paths

Field paths traverse your declared relations. Type a prefix inside the scope builder and the editor narrows to only valid continuations:

```ts twoslash
// @noErrors
import { engine } from "./engine";
// ---cut-before---
engine.defineResource("orders", {
  relations: {
    customer: true,
    orderLines: { with: { product: true } },
  },
  query: {
    scope: (f, ctx) => f.is("orderLines.", ctx.orgId),
    //                              ^|
  },
});
```

## Typed context

`.withContext<T>()` locks the `context` shape on every call — `query()` and `scope` both receive `ctx` as that exact type, no casting:

```ts twoslash
// @noErrors
import { createQueryEngine } from "drizzle-resource";
import { db, relations, schema } from "./twoslash-support";
// ---cut-before---
const engine = createQueryEngine({ db, schema, relations }).withContext<{
  orgId: string;
  userId: string;
}>();

engine.defineResource("orders", {
  query: {
    scope: (f, ctx) => f.is("status", ctx.orgId),
  },
});
```

## Compile-time errors for invalid paths

Any path not reachable through your declared `relations` tree is rejected at compile time:

```ts twoslash
// @errors: 2345
import { engine } from "./engine";
// ---cut-before---
engine.defineResource("orders", {
  relations: { customer: true },
  query: {
    scope: (f, ctx) => f.is("nonExistentField", ctx.orgId),
  },
});
```
