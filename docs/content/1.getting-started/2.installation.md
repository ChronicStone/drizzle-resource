# Installation

## Requirements

- **Drizzle ORM** `>=1.0.0-beta` with relational queries enabled
- A configured Drizzle database instance that exposes `db.query.<table>.findMany(...)`
- Your generated schema and relations (from `drizzle-kit` or equivalent)

## Install the package

::code-group

```bash [npm]
npm install drizzle-resource
```

```bash [pnpm]
pnpm add drizzle-resource
```

```bash [bun]
bun add drizzle-resource
```

::

## What you need from your Drizzle setup

The engine requires three things from your existing setup:

```ts twoslash
import { createQueryEngine } from "drizzle-resource";
import { db, relations, schema } from "./twoslash-support";

const engine = createQueryEngine({
  db, // your Drizzle db instance
  schema, // your schema map (import * as schema from './schema')
  relations, // your relations map
});
```

**`db`** — the Drizzle database instance. It must expose relational queries at `db.query.<table>.findMany(...)`. The engine uses this for row hydration.

**`schema`** — your schema map keyed by table name. Used for field-path inference and SQL compilation.

**`relations`** — your Drizzle relations map, built with `defineRelations` from the same schema. Used for relation traversal, nested field-path validation, and many-to-many join compilation.

::note
If you are using `drizzle-kit` with `--dialect pg` or equivalent, your generated output already includes everything the engine needs. Point `schema` and `relations` at those generated files.
::

## TypeScript

The package ships with full TypeScript types and does not require any additional `@types/*` packages. Set `strict: true` in your `tsconfig.json` to get the strongest inference on field paths.
