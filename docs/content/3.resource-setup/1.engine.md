---
title: Engine
description: How to create and configure the query engine — the factory that binds to your Drizzle database and produces resources.
---

The engine is the factory. You create it once, bind it to your Drizzle setup, and use it to define all your resources.

## Creating an engine

```ts twoslash [engine.ts]
import { createQueryEngine } from "drizzle-resource";
import { db } from "./db";
import { schema } from "./schema";
import { relations } from "./relations";

export const engine = createQueryEngine({ db, schema, relations });
```

### `db`

Your Drizzle database instance. Must expose relational queries at `db.query.<table>.findMany(...)`. The engine uses this for row hydration in the automatic pipeline.

### `schema`

Your Drizzle schema map. Typically `import * as schema from './schema'` from your generated or hand-written schema file. Used for field-path inference and SQL compilation.

### `relations`

Your Drizzle relations definition, built with Drizzle's `defineRelations`. The engine uses this to:

- resolve nested field paths like `customer.name` or `orderLines.product.category`
- determine join paths for filters, search, sort, and facets
- validate that declared relation paths in `defineResource` exist in the schema

## Attaching per-request context

Most APIs need some per-request data — a tenant id, the current user, a session token. Attach it with `.withContext<T>()`:

```ts twoslash
// @noErrors
import { createQueryEngine } from "drizzle-resource";
import { db } from "./db";
import { relations } from "./relations";
import { schema } from "./schema";

export const engine = createQueryEngine({ db, schema, relations }).withContext<{
  orgId: string;
  userId: string;
}>();
```

The context type flows through to:

- `query.scope` functions — receives `ctx` as the first argument
- `strategy.*` overrides — receives `ctx` in the utils object

When you call `resource.query({ context, request })`, the `context` argument must match this type.

If your resources need no per-request data, omit `.withContext()`.

## Sharing the engine

Create one engine instance per database connection and export it. All your resources import and use the same engine:

```ts twoslash
// @noErrors
import { createQueryEngine } from "drizzle-resource";
import { db } from "./db";
import { relations } from "./relations";
import { schema } from "./schema";

// engine.ts
export const engine = createQueryEngine({ db, schema, relations })
  .withContext<{ orgId: string }>()

// orders.resource.ts
import { engine } from './engine'
export const ordersResource = engine.defineResource('orders', { ... })

// products.resource.ts
import { engine } from './engine'
export const productsResource = engine.defineResource('products', { ... })
```

## Defining a scope at the engine level

Most resources should define `query.scope(filters, context)` inline. `engine.defineScope(...)` is still available when you want to reuse the same scope callback across multiple resources. See [Scope](/resource-setup/scope) for details.
