# Types

All types are exported from the package root:

```ts
import type {
  QueryRequest,
  QueryResponse,
  // ...
} from 'drizzle-resource'
```

## Request types

### `QueryRequest`

The full shape of a query request. See [Query Contract → Request Shape](/query-contract/request-shape).

```ts
type QueryRequest = {
  pagination: { pageIndex: number; pageSize: number }
  sorting: Array<{ key: string; dir: 'asc' | 'desc' }>
  filters: QueryFilterGroup
  search: { value: string; fields: string[] }
  context: Record<string, unknown>
  facets?: QueryFacetRequest[]
}
```

### `QueryFacetRequest`

```ts
type QueryFacetRequest = {
  key: string
  mode: 'exclude-self' | 'include-self'
  limit: number
}
```

## Response types

### `QueryResponse<TRow>`

```ts
type QueryResponse<TRow> = {
  rows: TRow[]
  rowCount: number
  facets?: QueryFacetResult[]
}
```

### `QueryIdsResponse<TId>`

```ts
type QueryIdsResponse<TId = unknown> = {
  ids: TId[]
  rowCount: number
}
```

### `QueryFacetResult`

```ts
type QueryFacetResult = {
  key: string
  options: Array<{ value: unknown; count: number }>
  total: number
}
```

### `QueryFacetsResponse`

```ts
type QueryFacetsResponse = {
  facets: QueryFacetResult[]
}
```

## Filter types

### `QueryFilterCondition<TField>`

```ts
type QueryFilterCondition<TField = string> = {
  type: 'condition'
  key: TField
  operator: QueryFilterOperator
  value: unknown
}
```

### `QueryFilterGroup<TField>`

```ts
type QueryFilterGroup<TField = string> = {
  type: 'group'
  combinator: 'and' | 'or'
  children: QueryFilterNode<TField>[]
}
```

### `QueryFilterNode<TField>`

```ts
type QueryFilterNode<TField = string> =
  | QueryFilterCondition<TField>
  | QueryFilterGroup<TField>
```

### `QueryFilterOperator`

```ts
type QueryFilterOperator =
  | 'is'
  | 'isNot'
  | 'isAnyOf'
  | 'contains'
  | 'gt'
  | 'gte'
  | 'lt'
  | 'lte'
  | 'between'
  | 'before'
  | 'after'
```

### `QueryFilterInput<TField>`

Flexible input type accepted by the filter builder methods:

```ts
type QueryFilterInput<TField = string> =
  | QueryFilterNode<TField>
  | QueryFilterNode<TField>[]
  | null
  | undefined
```

## Engine types

### `QueryEngine<...>`

Returned by `createQueryEngine(...)`. The generic parameters are inferred from your database, schema, and relations.

### `QueryResource<...>`

The type of a defined resource, returned by `engine.defineResource(...)`. Exposes `query`, `queryIds`, `queryRows`, `queryFacets`.

### `QueryFieldPath<TSchema, TRelations, TRoot, TWith>`

A type-level field path — a union of all valid dotted field path strings for a given resource root and relation tree. This is what powers the compile-time validation of sort keys, filter keys, and facet keys.
