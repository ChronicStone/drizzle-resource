---
title: Resource Methods
description: The four public methods on a defined resource — query, queryIds, queryRows, queryFacets.
---

Every resource created with `engine.defineResource(...)` exposes four public methods.

## `resource.query(options)`

The full pipeline. Runs ids selection, row hydration, and optional facet resolution in one call.

```ts twoslash
// @noErrors
import { ordersResource } from "./twoslash-support";

const result = await ordersResource.query({
  context: { orgId: "acme" },
  request: {
    pagination: { pageIndex: 1, pageSize: 25 },
    sorting: [{ key: "createdAt", dir: "desc" }],
    search: { value: "", fields: [] },
    filters: [],
    context: {},
    facets: [{ key: "status", mode: "exclude-self", limit: 10 }],
  },
});

// result: { rows: TRow[]; rowCount: number; facets?: QueryFacetResult[] }
```

**Returns:** `{ rows, rowCount, facets? }`

- `rows` — the current page of hydrated rows, typed from your schema and declared relations
- `rowCount` — total rows matching the filters before pagination
- `facets` — present only when `request.facets` is non-empty

## `resource.queryIds(options)`

Runs only the ids stage — returns page ids and the total count without hydrating rows.

```ts twoslash
import { ordersResource, request } from "./twoslash-support";

const { ids, rowCount } = await ordersResource.queryIds({
  context: { orgId: "acme" },
  request,
});
```

**Returns:** `{ ids: TId[], rowCount: number }`

Use this when you want to:

- Cache page ids separately from row data
- Pass ids to a different service for hydration
- Batch id selection across multiple resources before hydrating

## `resource.queryRows(options)`

Hydrates a set of pre-ordered ids without running id selection again.

```ts twoslash
import { ids, ordersResource, request } from "./twoslash-support";

const rows = await ordersResource.queryRows({
  context: { orgId: "acme" },
  request,
  ids, // ordered ids from a previous queryIds call
});
```

**Returns:** `{ rows: TRow[] }`

The rows are returned in the same order as the input `ids` array. `queryRows` does not run a count query.

## `resource.queryFacets(options)`

Resolves facets independently from the page query.

```ts twoslash
import { ordersResource, request } from "./twoslash-support";

const { facets } = await ordersResource.queryFacets({
  context: { orgId: "acme" },
  request,
  facets: [{ key: "status", mode: "exclude-self", limit: 10 }],
});
```

**Returns:** `{ facets: QueryFacetResult[] }`

Use this when the facet sidebar loads separately from the main table — for example, on initial page load before the user enters a search.

## Return types

```ts twoslash
import type {
  QueryFacetResult,
  QueryFacetsResponse,
  QueryIdsResponse,
  QueryResponse,
} from "drizzle-resource";

type QueryResponseExample<TRow> = QueryResponse<TRow>;
type QueryIdsResponseExample<TId> = QueryIdsResponse<TId>;
type QueryFacetsResponseExample = QueryFacetsResponse;
type QueryFacetResultExample = QueryFacetResult;
```
