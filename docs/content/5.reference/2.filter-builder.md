# Filter Builder

`createQueryFilterBuilder()` provides a typed DSL for constructing filter trees in server-side code — scope functions, strategy overrides, and tests.

## Import and create

```ts
import { createQueryFilterBuilder } from "drizzle-resource";

const f = createQueryFilterBuilder<"status" | "customer.name" | "createdAt">();
```

The generic parameter is a union of valid field paths. When using the builder inside `query.scope(filters, context)`, the engine infers the valid paths from the resource — you typically don't need to pass the generic manually.

## Condition methods

All condition methods return a `QueryFilterCondition`:

```ts
f.is("status", "active");
f.isNot("status", "cancelled");
f.isAnyOf("status", ["pending", "processing"]);
f.contains("customer.name", "acme");
f.gt("totalAmount", 100);
f.gte("totalAmount", 100);
f.lt("totalAmount", 1000);
f.lte("totalAmount", 1000);
f.between("createdAt", ["2026-01-01", "2026-12-31"]);
f.before("createdAt", "2026-06-01"); // alias for lt
f.after("createdAt", "2026-01-01"); // alias for gt
```

## Group methods

```ts
// AND group — all children must match
f.and([f.is("status", "active"), f.gte("totalAmount", 100)]);

// OR group — at least one child must match
f.or([f.contains("customer.name", "acme"), f.contains("customer.name", "globex")]);
```

Groups can be nested arbitrarily deep:

```ts
f.and([
  f.isNot("status", "cancelled"),
  f.or([
    f.gte("totalAmount", 1000),
    f.and([f.contains("customer.name", "acme"), f.after("createdAt", "2026-01-01")]),
  ]),
]);
```

## Using in scope functions

```ts
scope: (f, ctx) => f.and([f.is("customer.orgId", ctx.orgId), f.is("deletedAt", null)]);
```

The `f` argument inside `query.scope` is pre-typed to the resource's valid field paths.

## Using in strategy overrides

```ts
strategy: {
  ids: async ({ request, utils }) => {
    const extraFilter = f.and([
      f.is('status', 'active'),
    ])
    const where = utils.buildWhereClause({
      filters: mergeFilters(request.filters, extraFilter),
      search: request.search,
    })
    // ...
  },
}
```

## Return types

```ts
type QueryFilterCondition<TField> = {
  type: "condition";
  key: TField;
  operator: QueryFilterOperator;
  value: unknown;
};

type QueryFilterGroup<TField> = {
  type: "group";
  combinator: "and" | "or";
  children: QueryFilterNode<TField>[];
};

type QueryFilterNode<TField> = QueryFilterCondition<TField> | QueryFilterGroup<TField>;
```
